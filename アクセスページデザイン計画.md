# アクセスページ デザイン計画

**基準:** Awwwards 8.0 を意識。現行の「2列テーブル＋地図」の凡庸な構成を避け、**トレンドを抑えた**デザインにする。  
**スキル:** Web Design Skill 2026年版 Part1（汎用原則・2026トレンド）を適用。

---

## 1. コンテキスト定義

| 項目 | 内容 |
|------|------|
| **ターゲット** | 県連に連絡・訪問したい有権者、党員・サポーター、取材・問い合わせ希望者 |
| **目的** | 所在地・連絡先を確実に伝え、地図で場所を把握しやすくする。問い合わせフォームへの導線を明確にする。 |
| **コンテンツ** | 名称・郵便番号・住所・電話・FAX・メール（お問い合わせフォームへのリンク）、Google地図埋め込み |

---

## 2. 現行デザインの課題（避けるべき）

- **均等な2列テーブル** → 単調で「テンプレート品質」。プレミアムには非対称・情報の階層化が必要。
- **地図と情報が同列** → スケール差がなく、視線の焦点がぼやける。
- **余白不足** → セクション間が詰まっているとチープに見える。
- **CTAが弱い** → 「お問い合わせフォーム」がテキストリンクだけだと行動に結びつきにくい。

---

## 3. デザイン方針（トレンドを抑える）

### 3-1. レイアウト — 非対称 60:40

- **左 40%**: 連絡先情報（カード or リスト）。コンパクトにまとめ、見出し＋行単位の情報。
- **右 60%**: 地図を**大きく**配置。画像とテキストのスケール差で「地図が主役」にする。
- 同一パターンは1ページ2回まで。このページは「情報ブロック＋地図」の1パターンで完結。

### 3-2. Bento の考え方（情報の塊）

- 連絡先を「1枚のカード」にまとめる（影で浮かせる、角丸 8px）。中はラベル＋値のリスト形式だが、**table のグレー線は使わない**。行間と背景の差で分離。
- 地図は別ブロック。角丸 12px、アスペクト比 4:3 または 16:9 で統一感。
- その下に「お問い合わせフォームへ」の **CTA ボタン**（アクセント1色、ホバーでリフト＋グロー）。

### 3-3. 余白・スペーシング（8pxグリッド）

- セクション間: **120px〜160px**。
- コンテンツ幅: max **1200px**。左右パディング: `clamp(24px, 5vw, 80px)`。
- カード内 padding: **32px**。関連要素は 16px、非関連は 48px。

### 3-4. タイポグラフィ

- ページ見出し（h1）: `clamp(24px, 3.5vw, 36px)` / weight 400〜500 / letter-spacing 0.06em。
- カード内ラベル: 12〜13px / weight 500 / 色は --text-muted。
- 値（住所・電話など）: 15〜17px / weight 400。電話は `tel:` リンク化。
- 和文 font-weight は **最大500**。

### 3-5. カラー（色の引き算）

- 既存トークン: --navy, --accent, --bg-page: #FAFAF7, --text: #1C1C1C, --text-muted。
- 使用比率: ニュートラル **80%** / ネイビー **15%** / アクセント **5%**。
- アクセントは **CTA ボタン「お問い合わせフォームへ」のみ**。他はネイビー・グレーで統一。

### 3-6. アニメーション

- リード文・連絡先カード・地図・CTA を **スクロールイン**（opacity + translateY(16px→0)、600ms、cubic-bezier(0.16, 1, 0.3, 1)、スタガー 50ms）。
- CTA ボタン: ホバーで `translateY(-2px)` ＋ 軽い box-shadow（accent のグロー）。transition 400ms。
- `prefers-reduced-motion: reduce` で無効化。

### 3-7. 禁止事項（スキル準拠）

- `border: 1px solid #ccc` のグレー線 → 使わない。影と背景で分離。
- 純白 #FFFFFF 背景 → オフホワイト。
- 角丸 999px（ピル型）→ 8px または 12px。
- 同一レイアウトパターン3回以上、均等グリッドの連続。

---

## 4. 構成（上から）

1. **ヘッダー** — 既存共通
2. **セクションバー** — 「アクセス」
3. **リード** — 1行（例:「奈良県連の所在地・連絡先です。お気軽にお問い合わせください。」）
4. **メイン 2カラム（非対称）**
   - 左: 連絡先カード（名称・郵便番号・住所・電話・FAX・メール＝お問い合わせフォームへのリンク）
   - 右: Google 地図（iframe、角丸・影）
5. **CTA** — 「お問い合わせフォーム」へのボタン（access の下、またはカード内のメール行をボタン化して1つにしても可）
6. **フッター** — 既存共通
