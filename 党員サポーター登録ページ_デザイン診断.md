# 党員・サポーター登録申し込みフォーム — デザイン診断レポート

**診断日**: 2026年3月  
**対象**: 党員・サポーター登録申し込みフォーム（参考サイト new-kokumin-nara.jp/supporter/）  
**基準**: Design Critic スキル（Awwwards審査員視点・5軸採点）

## 致命的な問題

🔴 **1. フォーム要素が「入力しづらい・信頼しづらい」と感じさせる**  
入力欄のボーダーが極端に薄く背景と同化している。ラジオは四角枠で一般的な円形ラジオでなく、どの選択肢が選ばれているか分かりにくい。フォーカス・ホバー・エラー状態のスタイルが不明瞭で、**「ちゃんとしたフォーム」に見えず、離脱・未記入の原因になる。**

🔴 **2. 注意枠の赤・詰め込みで心理的負荷が高い**  
【注意】を赤枠・赤文字・小さいフォントで詰め込んでいると、**最初に目に入るのがリスク情報**になり、「面倒・怖い」と感じさせる。重要な注意は残しつつ、色・余白・階層で「読める・落ち着ける」形にしないと、申し込み意欲を削ぐ。

🔴 **3. CTA（送信ボタン）が弱く、完了への一押しが伝わらない**  
送信ボタンが淡い黄・地味なスタイルで、**「ここを押せば申し込み完了」**という役割が視覚的に伝わらない。フォームの主目的であるCVを高めるには、サイズ・色・ラベル・余白で「唯一の主行動」として際立たせる必要がある。

## 改善指示（優先順位順）

🔴 **1. [致命的] 入力欄を「触れる・選べる」と分かるデザインにする**  
- **問題**: ボーダーが薄く、入力欄と背景の区別がつきにくい。  
- **修正**: `border: 1px solid #E0E0E0` → `border: 2px solid #ccc`、背景は `#FAFAFA`。フォーカス時は `border-color: var(--navy)` と `box-shadow: 0 0 0 3px rgba(43,57,144,0.15)`。  
- **理由**: 入力可能領域が明確になり、離脱・入力ミスを減らせる。

🔴 **2. [致命的] ラジオ・選択UIを標準的で分かりやすい形にする**  
- **問題**: 党員/サポーター・性別が四角枠で、ラジオとして認識しづらい。  
- **修正**: 円形の `input type="radio"` をスタイルし、`accent-color: var(--navy)`。ラベルはタップ領域を含め `padding: 12px 16px`、`min-height: 44px` でタッチターゲットを確保。  
- **理由**: 誰でも「選ぶ」操作だと理解でき、アクセシビリティとCVRが上がる。

🔴 **3. [致命的] 送信ボタンを主CTAとして際立たせる**  
- **問題**: 色・サイズが弱く、最後の一押しが伝わらない。  
- **修正**: 背景 `var(--navy)` または `var(--accent)`、文字は白、`min-height: 52px`、`padding: 16px 48px`、`font-size: 17px`、`font-weight: 700`。角丸は `8px`。ホバーで `opacity: 0.9` または `transform: translateY(-1px)`。  
- **理由**: 「ここを押せば申し込み完了」が一目で分かり、送信率が上がる。

🟡 **4. [重要] 注意ブロックを「読める・落ち着ける」デザインに変える**  
- **問題**: 赤枠・赤文字・小字の詰め込みで心理的負荷が高い。  
- **修正**: 枠は `border-left: 4px solid #C62828`、背景は `#FFF8F8`、文字は本文色 `#333` にし、見出しだけ「注意」を `color: #B71C1C`。`padding: 24px`、行間 `1.8`、フォントは `14px` 以上。  
- **理由**: 注意は伝えつつ、恐怖感を減らしフォーム全体の信頼感を上げる。

🟡 **5. [重要] セクション間・ラベルと入力の余白をルール化する**  
- **問題**: 要素が詰まって読みづらく、安っぽく見える。  
- **修正**: フォームブロック間は `margin-bottom: 32px`。ラベルと入力の間は `8px`、入力と次のラベルの間は `24px`。タイトルバー下〜最初の項目は `48px`。  
- **理由**: 余白のルールで「丁寧なフォーム」と感じさせ、離脱を減らす。

🟡 **6. [重要] 背景を純白からオフホワイトに変える**  
- **問題**: `#FFFFFF` 単色だと安っぽく、入力欄との区別も弱い。  
- **修正**: `body` またはメインエリアを `#FAFAFA` や `#F5F5F5` に。フォームカードがあればその中だけ `#FFFFFF` でメリハリ。  
- **理由**: 背景と入力エリアの関係がはっきりし、美しさ・信頼感が増す。

🟢 **7. [細部] タイトルバー下に短いリード文を足す**  
- **問題**: 青バーの直後に注意・フォームが続き、ベネフィットが伝わらない。  
- **修正**: 「奈良から、国民民主党を支える。ご登録は以下のフォームから。」など1行を、`font-size: 15px`、`color: #555`、`margin-bottom: 32px` で追加。  
- **理由**: 目的とベネフィットが先に伝わり、フォームへの心理的ハードルが下がる。

🟢 **8. [細部] 会費の注意（黄ハイライト）の視認性を上げる**  
- **問題**: 送信ボタン下の黄背景が単純で、重要情報としての階層が弱い。  
- **修正**: 背景 `#FFFDE7`、左ボーダー `4px solid var(--accent)`、`padding: 16px 20px`、`font-size: 14px`。  
- **理由**: 送信後の次のアクションが分かり、サポート問い合わせを減らせる。

🟢 **9. [細部] 必須マーク（※）のスタイルを統一する**  
- **問題**: ※ がそのまま並ぶと雑に見える。  
- **修正**: `<span class="required">※</span>` で `color: var(--navy)` または `#C62828`、`font-size: 12px`、`margin-left: 2px`。  
- **理由**: 必須項目が分かりやすくなり、入力漏れ防止と見た目の一貫性が両立する。

🟢 **10. [細部] ヘッダーのリンク・電話・お問い合わせをタッチしやすくする**  
- **問題**: アイコンとテキストが小さく、SPで押しづらい。  
- **修正**: タップ領域を `min-height: 44px`、`padding: 8px 12px` で確保。  
- **理由**: アクセシビリティと使いやすさが向上する。

## 唯一の良い点

**「党員・サポーター登録申し込みフォーム」という目的がタイトルバーで明示されており、何のページかは一応理解できる。** ただし、見た目・フォームUI・注意の出し方で信頼と完了率を大きく損なっているため、上記の致命的な3点と重要3点から手を入れることを強く推奨する。
