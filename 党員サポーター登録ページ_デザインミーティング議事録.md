# 党員・サポーター登録ページ — デザインミーティング議事録

**日付**: 2026年3月  
**参照**: 設計書（スコア目標と設計.md）、デザインスキル（design-system / anti-ai-aesthetic）  
**制約**: **国民民主党のカラーを逸脱しない。** 使用する色は既存LPのトークン（--navy, --accent, --bg-white 等）に限定する。

---

## 1. ミーティングの目的

設計書をデザインスキルの観点で見直し、75点（B+）達成と「国民民主党らしさ」の両立を図る。  
**絶対条件**: 党色以外の新色（赤・緑・他ブランド色）は一切使わない。

---

## 2. カラー方針の確定（党色厳守）

### 2-1. 使用を許容する色のみ列挙

| 用途 | 変数・値 | 備考 |
|------|----------|------|
| メイン・CTA・タイトルバー・フォーカス | `--navy` #2B3990 | 主役。送信ボタン・セクションバー・入力フォーカスリング |
| 濃いネイビー（ホバー等） | `--navy-dark` #1E2D6F | ボタンホバーで使用可 |
| アクセント（補助のみ） | `--accent` #F5A623 | 使用は「ごく一部」に限定（後述） |
| 背景（ページ） | `--bg-page` = #FAFAFA | 既存LPの --bg-gray に近いオフホワイト |
| 背景（カード・入力） | `--bg-white` #FFFFFF | フォームカード・input 背景 |
| テキスト | `--text` #333333 または #222222 | 既存LPに合わせる |
| 補助テキスト | `--text-date` / `--text-muted` #666666 | ラベル・注意の補足文 |
| ボーダー | `--border` #E0E0E0 | 入力欄・カードの境界 |

### 2-2. 使用を禁止する色

- **赤系**（#E57373, #C62828 等）: 注意ブロックに従来使われがちだが、**党色にないため使用禁止**。注意はネイビーまたはグレーで表現する。
- その他、既存トークンにない新色は一切使わない。

### 2-3. 注意ブロックの色（ブラッシュアップ結論）

- **背景**: `#F5F6FA`（ネイビーを極薄にしたニュートラル）。または `#FAFAFA`。
- **左ボーダー**: `4px solid rgba(43, 57, 144, 0.4)`（--navy の 40%）。「注意」であることは伝えつつ、党色に統一。
- **見出し「注意」**: `color: var(--navy)`、`font-weight: 700`。
- **本文**: `color: var(--text)`。赤文字は使わない。

### 2-4. 送信後案内ブロック（会費の知らせ）の色

- **背景**: `rgba(245, 166, 35, 0.06)`（--accent の 6%）。黄系だが党のアクセント色の派生に限定。
- **左ボーダー**: `4px solid var(--accent)`。
- これにより「読んでほしい補足」が視覚的に分かりつつ、党色のみで完結する。

### 2-5. アクセントの役割分担（1ページ1主役）

- **主役**: ネイビー。送信ボタン・タイトルバー・フォーカスリング・注意の見出し。
- **脇役**: オレンジ（--accent）。送信後案内の左ライン・必須マークの小さなドット等、**面積はごく少なく**。
- ネイビーとオレンジの「2色」は党の公式トークンなので許容するが、**赤・緑・他は入れない**。

## 3. 余白・タイポの数値確定（デザインスキル反映）

### 3-1. 余白ルール（8pxグリッド・密度のコントラスト）

| 箇所 | 値 | 根拠 |
|------|-----|------|
| タイトルバー 〜 リード | 64px | セクション間。見出しの上を効かせる。 |
| リード 〜 注意ブロック | 48px | 関連がやや近い。 |
| 注意ブロック 〜 フォームカード | 64px | 非関連は遠く。 |
| フォーム内：ラベル 〜 入力 | 8px | 関連は近く。 |
| フォーム内：入力 〜 次ラベル | 24px | 要素間の息継ぎ。 |
| 送信ボタン上マージン | 48px | CTA の前に一呼吸。 |
| フォームカード内 padding | 32px（SP 24px） | カード内は 32〜48px の範囲。 |

### 3-2. タイポグラフィ（和文 weight 500 以下・ジャンプ率）

| 要素 | 指定 | 備考 |
|------|------|------|
| タイトルバー | 18px / 700 / 0.08em | 既存 .section-bar に合わせる。 |
| リード | clamp(15px, 2vw, 17px) / 500 / line-height 2 | 本文よりやや大きく、重くしすぎない。 |
| 注意ブロック見出し | 14px / 700 / var(--navy) | 本文より少し強く。 |
| 注意ブロック本文 | 14px / 400 / line-height 1.8 | 可読性優先。 |
| フォームラベル | 14px / 500 / var(--text) | 必須マークは 12px・var(--navy)。 |
| 入力 placeholder | 14px / 400 / var(--text-muted) | |
| 送信ボタン | 17px / 700 | 主CTAなのではっきり。 |

---

## 4. コンポーネント別の仕様確定

### 4-1. フォームカード

- **背景**: #FFFFFF。
- **角丸**: 8px（設計書どおり）。anti-ai の「ゆらぎ」で、ボタンだけ 10px にして差を付けても可（任意）。
- **影**: `0 2px 8px rgba(0,0,0,0.06)`。境界をはっきりさせすぎない。
- **最大幅**: 560px。中央配置。

### 4-2. 入力欄

- **ボーダー**: 2px solid #E0E0E0。フォーカス時: `2px solid var(--navy)` + `box-shadow: 0 0 0 3px rgba(43,57,144,0.12)`。
- **背景**: #FFFFFF。角丸 8px。
- **padding**: 14px 16px。min-height 48px（タッチ 44px 以上）。

### 4-3. ラジオボタン

- 標準の円形に戻す。`accent-color: var(--navy)` で党色に統一。
- ラベルはクリック領域を含め padding 12px 16px、min-height 44px。

### 4-4. 送信ボタン（主CTA）

- **背景**: `var(--navy)`。ホバー: `var(--navy-dark)`。
- **文字**: 白、17px、700。
- **高さ**: 52px。角丸 10px（カード 8px とのゆらぎ）。
- **影（ホバー）**: `0 4px 14px rgba(43,57,144,0.2)`。ネイビーを含めた影でブランド一貫。

---

## 5. インタラクション（楽しさ 12/20 達成）

- **送信ボタン**: ホバーで `translateY(-2px)` + 上記シャドウ。transition 300ms ease。
- **入力**: focus でボーダー・グロー。`outline: none` の代わりに box-shadow でフォーカスリング。
- **ページ表示**: タイトルバー → リード → 注意 → フォームカードを 0.1s スタガーで fade-in。duration 400ms。`prefers-reduced-motion: reduce` のときはアニメなし。

## 7. まとめ（ブラッシュアップの要点）

| 項目 | 内容 |
|------|------|
| **カラー** | 国民民主党トークンのみ。赤は使わない。注意はネイビー系、送信後案内はアクセントの薄い派生。 |
| **主役色** | ネイビー（CTA・タイトル・フォーカス）。オレンジは補助に限定。 |
| **余白** | 8px グリッド、セクション間 48〜64px、カード内 32px。 |
| **タイポ** | 和文 500 以下、リード 17px・本文 15px・ラベル 14px でジャンプ率を付ける。 |
| **一貫性** | 角丸 8px（ボタンは 10px 可）。影はネイビーを含めてブランド感を統一。 |

この議事録と設計書 v1.1 に基づき実装すれば、党色を逸脱せず 75 点（B+）を狙える設計となる。
